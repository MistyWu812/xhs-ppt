You turn an article/draft into PNG 卡片 (Xiaohongshu / social cards). **必须按三段式流程走，不要拿到内容就直接写 HTML。** 第二步出分镜给用户确认后，第三步才渲染。

> **北极星：只看 PPT 就能理解原文的核心意思。** 卡片不是预告片，是原文的「压缩版」——读者不点开原文，光刷完这套卡，就要拿到文章的完整论点、关键证据、结论。三步都围绕这个目标：第一步把每段核心抽成完整金句（不是钩子），第二步保证页集覆盖原文所有要点（不漏不只是"点到为止"），第三步做「脱稿自检」。

输入：用户粘贴的原文、一个文章/markdown 文件路径、或一段文本。可选提示：配色 / 比例 / 是否 2合1（没说就按第一步自己判，真歧义才问）。

**路径约定**：本 skill 自带 `templates/` 和 `tools/`，均以本 skill 根目录为基准引用（下文记作 `<SKILL>`）。渲染产物写到用户指定的输出目录（下文记作 `<OUT>/<slug>/`，没指定就用当前目录下的 `out/<slug>/`）。

# 第一步 · 读原文 → 判属性 → 定配色 + 配图

1. **读全文 + 抽「核心点清单」**。先列出原文**所有核心论点**（每个 section 的主张 + 关键证据/数字 + 结论），这份清单是后面"不漏"的对照表。再把每个核心点提炼成**完整金句**——是能独立读懂的结论句（如「Builder Pool 让员工从卖时间变卖资产」），不是钩子（如「薪酬有个反常识设计」）。卡片只上金句，绝不堆原文长段，但金句必须自带结论。
2. **判文章属性**（决定整套气质）：
   | 属性 | 特征 | 典型 |
   |---|---|---|
   | 观点/认知重构 | 有暴论、反常识、立场强 | 「AI 时代 HR 重新分层」 |
   | 方法论/体系 | 多步骤、有框架、可落地 | 「30/60/90 落地路径」 |
   | 案例/对比拆解 | 多主体并列、横向比 | 「3 家 AI 原生公司」 |
   | 复盘/避坑 | 教训、红线、负向警示 | 「招聘 5 个坑」 |
   | 清单/工具 | 并列要点、checklist | 「10 个 Prompt 模板」 |
   | 故事/情绪 | 第一人称、有温度 | 「我裸辞那年」 |
3. **定配色**（属性 → 配色，整套统一不混用）：
   | 属性 | 首选配色 | 理由 |
   |---|---|---|
   | 观点强 / AI·产品向 / 体系·分层 | **科技深色·亮电光青**（旗舰） | 质感高、级别徽章/数字最出彩 |
   | 同上但需离线 / 更稳重 | **深蓝**（纯 SVG 图标，离线安全） | 不依赖 Web 字体 |
   | HR 方法论 / 清单 / 正向温和 | **专业蓝绿**（浅色） | HR/企业主最买账 |
   | 故事/情绪 | **暖中性** | 有温度 |
   | 单点暴论 / 极简 | **极简黑** | 强冲击 |
   （token 见后文「配色库」。默认走科技深色·亮电光青。）
4. **定配图**（= 图标体系 + 图示词汇，不是照片）：
   - **图标体系**：联网→Material Icons（旗舰）；离线→内联 SVG（feather 风）。
   - **图示词汇** = 这套要用到的图表类型，先圈定，第二步再分配到页：Hero 大数字 / ✗vs✓对比卡 / 流程卡(箭头链) / 金字塔·阶梯 / 全景表 / 环形嵌套 / 2×2 总结 / 图标卡网格。
5. **产出**：一句话说明「这是 X 属性 → 用 Y 配色 + Z 图标，主打这几类图示」。

# 第二步 · 设计框架（先出分镜，别动 HTML）

目标：定**页数、每页结构、哪页用流程图、哪页用组合图**，写成分镜让用户确认。

1. **定页数**：小红书 6–10 页较稳（封面 1 + 内页 4–7 + 结尾 1）。一段核心信息 = 一页，宁可拆页也别挤。2合1 出图时页数最好凑偶数（便于两两拼）。
2. **排叙事弧**（按属性选骨架）：
   - 观点/体系：封面钩子 → 为什么(问题/旧✗新✓) → 总览图 → 逐项展开(每项一页) → 关键/红线 → 总结升华。
   - 案例/对比：封面 → 为什么看 → 逐个案例(每个一页) → 横向对比表 → 选型建议 → CTA。
   - 复盘/清单：封面 → 总纲 → 逐条(坑/项) → 强调条 → checklist 结尾。
3. **每页选图示**（内容逻辑 → 组件，这步回答「哪里流程图、哪里组合图」）：
   | 内容逻辑 | 用什么图 | 组件 |
   |---|---|---|
   | 因果 / 时间 / 演化 / 回路 | **流程图** | ⑥ 流程卡(卡→箭头→卡) |
   | 层级 / 递进 / 嵌套结构 | 金字塔·阶梯 / 环形 | ⑤ 阶梯、环形嵌套 |
   | 新旧·对错·两条路 | ✗vs✓ 对比卡 | ③ |
   | 一个概念多面展开（定义+数字+清单+对比同页） | **组合图** | ② 左右分栏多组件拼 |
   | 多主体 × 多维度全量比 | 全景表 | ⑦ |
   | 单个震撼数字/结论 | Hero 数字 | 大字卡 |
   | 总分收尾 | 2×2 图标卡 + 金句条 | ⑧ |
   - **流程图** = 表达"先后/因果/链路"，用箭头串起 3–4 个卡（如 今天→明天→习惯、执行→抽查→兜底）。
   - **组合图** = 一页里拼 ≥2 个组件讲清一个主题（如 左「定义卡+Hero数字」/ 右「图标网格+对比卡」，或 环形图+三层注解）。信息密但要留白，别糊。
4. **覆盖性校验**：拿第一步的「核心点清单」逐条勾，确认每个核心论点都落到了某一页——漏了就加页或并入。宁可多一页，也别让读者读完缺一块。
5. **写分镜**到 `<OUT>/<slug>/_storyboard.md`，一行一页：`页码 | 区块名 | 核心金句 | 组件/图示 | 备注`；末尾附「核心点 → 落在第几页」对照表。参考样例：`examples/hr-collab-modes/_storyboard.md`。
6. **把分镜贴给用户确认**（页数对不对、有没有漏要点、哪页该上流程图/组合图、金句准不准）。**确认后才进第三步。**

# 第三步 · 输出 PPT（拷样张 → 填组件 → 渲染 → 拼接）

**起点永远是已验收样张，不从零写。** 旗舰源在 `<SKILL>/templates/tech-cyan/`：`_skeleton.html`(单页拷贝起点)、`_starter.py`(参数化批量生成起手，含全套组件函数)、`_components.html`(九大组件真实片段)、`slide01–11.html`(整套范例)、`_sample-L2.png`(样张)。出图前先 Read 一张已验收 slide 对齐手感。

1. 建目录 `<OUT>/<slug>/`。
2. 按分镜，每页 `cp <SKILL>/templates/tech-cyan/_skeleton.html` → 改名（零填充，命名决定渲染+拼接顺序）：
   - 16:9 / 2合1：`slide01.html…slideNN.html`（配 `stack_cards.py --pairs`）
   - 9:16：`00-cover.html` / `01.html…` / `99-cta.html`
   - 内容多变、参数化强（多模式/多层/多案例）时，**首选 `cp <SKILL>/templates/tech-cyan/_starter.py` 到 `<OUT>/<slug>/_gen.py`** 批量产 HTML——里面已带 cover()/modepage()/classbar()/relaystep()/evo()/trow()/pitrow() 全套组件函数 + ACC 色编码，改最底部「你的 slides」区即可。完整范例见 `examples/hr-collab-modes/_gen.py`。
3. 从 `_components.html` 抠分镜指定的组件填 `<main>`，只改文案/页码，**不动 `<style>` token**。
4. 渲染（§渲染）。2合1：先出 16:9 单页，再 stack（§2合1）。
5. **自检每页 PNG**：Read 每张渲染图，确认无文字溢出、配色统一、图标/字体已加载；溢出就压字数或拆页，重渲染。
6. **脱稿自检（北极星验收）**：只看渲染出的卡片（假装没读过原文），逐张过——能不能复述出原文的完整论点链？对照第一步「核心点清单」，哪条只看卡看不懂、或缺了，就回去把那页金句补成完整结论、或补图示/补页。读者只看卡 = 读懂原文，才算过。
7. 交付：说明渲染了 N 张、路径、用了什么配色/比例、脱稿自检结论。

# 配色库（token，照抄即得观感）

### 科技深色·亮电光青（旗舰，全内联在 `_skeleton.html`）
- **底** `#1a1a2e` + 右上青辉光 `radial(rgba(0,217,255,.14))` + 左下紫辉光 `radial(rgba(168,85,247,.12))` + 48px 青网格 `.grid`(radial mask 只露上中部)。
- **主青** `#00d9ff`｜**橙** `#ff9f00→#ff7a00`｜紫辉光 `#a855f7`｜正文 `#e8ecf5`/次级 `#aab4c8`/灰 `#9aa7bd`。
- **级别色编码**（统一别乱用）：L3=橙渐变｜L2=青蓝 `#00d9ff→#3b82e6`｜L1=青描边｜L0=灰描边。橙=关键/红线，青=正向/价值。
- **字**：中文 Noto Sans SC；数字/拉丁/徽章用 **Orbitron**(`.title-font`)。**需联网**加载 Google Fonts + Material Icons。
- 卡 `rgba(0,217,255,.06)`底+`.2`边+14px 圆角；kicker 胶囊；header=kicker+`h1.t`(关键词 `.o` 上橙)+`.rule` 下划线；pageno 右下。

### 备选
- **深蓝**（深色·离线安全·纯 SVG）：BG `linear(#0a1020,#0c1326)` / 主青 `#4fcfe0` / banner `linear(90deg,#178fb4,#2360c9)`+橙竖条。样张 `<SKILL>/templates/tech-navy/`。
- **专业蓝绿** ⭐（浅色·HR 主力）：BG `#FFF/#F7F9FC` / FG `#1A2740` / 主蓝 `#2B6FD8` / 绿 `#10B981` / 负向粉红 `#FDECEC+#E0535A`。
- **暖中性** `#FAF6F0/#2A2018/#D97A2C`；**极简黑** `#111/#FFF/#FFC857`。

# 比例（选一种）
| 比例 | 尺寸 | 用途 |
|---|---|---|
| 9:16 竖卡 | 1080×1440 | 小红书信息流，默认 |
| 16:9 单页 | 1280×720 | 一页一图，导 Keynote/PPT |
| 2合1 竖卡 | 1280×1440 | 两个 16:9 上下拼（公众号/小红书长图） |
一律 **@2x**(scale=2)。

# 渲染
```bash
python3 <SKILL>/tools/render_cards.py <slides_dir> 1280x720  2   # 16:9 单页
python3 <SKILL>/tools/render_cards.py <slides_dir> 1080x1440 2   # 9:16 竖卡
python3 <SKILL>/tools/render_cards.py <slides_dir> 1280x1440 2   # 单 HTML 含两屏直出 2合1
```
脚本已带 `--force-device-scale-factor`、`--virtual-time-budget`(等 Web 字体)、`--run-all-compositor-stages-before-draw`。检查 PNG 数==HTML 数；失败读 stderr。需要 Google Chrome（macOS 自动找；其它平台用 `CHROME=/path/to/chrome` 覆盖）。

# 2合1 拼接
```bash
BG=#1a1a2e python3 <SKILL>/tools/stack_cards.py --pairs <slides_dir> 2up   # 1+2,3+4,…→2up-N.png
```
科技深色底 `BG=#1a1a2e`；深蓝底 `BG=#0a1020`。奇数尾页原样保留。需要 `pip install pillow`。

# 长图文（整篇=一张超长图）
不切多卡，用 `render_longform.py`(HTML 内写 `<!-- render-size: 1080x4500 -->`)。版式 = 封面胶囊→粗标题→对比卡→图标网格→强调条→checklist。

# 红线
- **流程不能跳**：拿到内容先判属性定配色(一)→出分镜确认(二)→才渲染(三)。不要省略分镜直接出图。
- 文字不溢出、每页字数严控；**配色不混用、级别色编码不乱用**(L3 永远橙、L0 永远灰)。
- 一套图 = 一种比例 + 一套配色 + 一致封面。
- 联网才用 Web 字体(Orbitron/Material Icons)且必须 @2x + 字体等待；离线回退系统字 + SVG + 深蓝。
- 卡片要能脱离原文独立读懂——每页给完整结论，不是点到为止；但靠金句+图示压缩，绝不堆长段。
- 新卡从 `_skeleton.html` 拷，不从零写，不动共享 `<style>` token。
